# Introduction

Large language models (LLMs) have become a default interface for a wide range of language tasks, from factual question answering to multi-step reasoning and code generation. In practice, however, no single LLM is optimal for every query: models differ in capability, latency, and monetary cost. Always calling a frontier model maximizes quality but is often unnecessary for easy queries; always calling a small model reduces cost but fails when the query requires stronger reasoning. This tension has motivated **LLM routing** (also called **query routing**): given an input query, select which model from a candidate pool should answer it so as to preserve response quality while reducing cost and related serving overhead.

A growing body of work addresses this problem in a **supervised** manner. RouteLLM learns a binary weak/strong router from human preference data and related labeled augmentations, predicting the probability that a strong model wins on a query and thresholding that score against a cost budget \cite{ong2025routellm}. Hybrid LLM likewise trains a binary router, but constructs labels from estimated quality gaps between a small and a large model using repeated generations scored by an automatic metric \cite{ding2024hybrid}. Subsequent systems extend the setting beyond a fixed pair: IRT-Router casts multi-LLM routing as a psychometric ability--difficulty interaction over dense query--model performance labels \cite{song2025irt}; MixLLM combines quality, cost, and latency in an online bandit-style decision rule over per-model predictors \cite{wang2025mixllm}; and ICL-Router represents each model's capability as in-context profiles so that new models can be added with limited profiling rather than full router retraining \cite{wang2025icl}. Related work further replaces preference or accuracy labels with uncertainty-derived winners---for example, semantic entropy used to construct training preferences for otherwise standard predictive routers \cite{zhang2025uncertainty}. Across these lines, the dominant pattern is the same: routing decisions are learned from **prior labeled outcomes** (preferences, quality gaps, correctness matrices, or uncertainty-derived winners), and the trained router then maps a new query to a model.

This supervised framing leaves an important gap. Building such labels requires large preference collections, full query--model evaluation matrices, or expensive multi-sample generation for every training example. More fundamentally, the router is trained under the assumption that one already knows---or can approximate---which queries should go where. What remains less explored is whether routing can be driven by **signals estimated from the query and from candidate models themselves**, without treating labeled query-to-model assignments as the primary starting point. We refer to this setting as **unsupervised LLM routing**: given a query, estimate a feature vector of routing signals and use those signals to decide which LLM to call. A second gap concerns *when* the decision is made. Many practical systems effectively rely on post-execution behavior---running a model or cascade and then correcting, retrying, or escalating. In contrast, we focus on a **pre-inference** decision: judging, before committing to full task execution, how much complexity or reasoning a query appears to require and which model is appropriate for it.

To address these gaps, we study unsupervised routing to LLMs through explicitly defined routing signals, followed by optional parameter weighting over those signals. Concretely, we distinguish:

- **Model-independent signals**, such as query or task complexity, which characterize the input without reference to a particular LLM; and
- **Model-dependent signals**, which characterize the interaction between a query and a candidate model. Following the discussion above, these include the entropy of the query with respect to a model \(M_i\), and **paraphrase-based uncertainty**: the stability of a model's answers when the same information need is expressed in multiple surface forms. A model that remains consistent under rephrasing is treated as more reliable for that query than a model whose answers change under equivalent asks.

These signals form a feature vector for each query (and, for model-dependent terms, for each query--model pair). Routing can then proceed in two ways: (i) **rule-based** assignment (for example, preferring a model with low uncertainty on the query), or (ii) **parameter weighting**, in which a small amount of labeled data is used only to learn combination weights over the unsupervised signals, rather than to learn the routing mapping from scratch from preference- or outcome-scale supervision. Our present scope is LLM routing; extension to agents---where an agent is viewed as orchestration plus an LLM---is left as future work, so that the core claim remains comparable to existing LLM routing literature.

**Contributions.** This paper makes the following contributions:

\begin{enumerate}
\item We formulate **unsupervised LLM routing** as the problem of selecting a model for a query from estimated routing signals, in contrast to the dominant supervised paradigm that learns routers primarily from preference, quality-gap, or correctness labels \cite{ong2025routellm,ding2024hybrid,song2025irt,wang2025mixllm,wang2025icl,zhang2025uncertainty}.
\item We organize routing features into **model-independent** and **model-dependent** signals, and instantiate the latter with model-conditional entropy and paraphrase-based uncertainty, aimed at capturing not only query difficulty but also a model's robustness to a given query.
\item We study how these signals can be used either through **decision rules** or through **parameter weighting**, and evaluate---including the possibility of weak or negative correlations---how such features relate to routing quality under a pre-inference decision framing.
\end{enumerate}
